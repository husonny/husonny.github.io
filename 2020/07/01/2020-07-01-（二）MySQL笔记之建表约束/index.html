（二）MySQL笔记之建表约束

Posted by Sonny on 2020-07-01

1.主键约束

它能够唯一确定一张表中的一条记录，也就是我们通过某个字段添加约束，就可以是的改字段不重复且不为空。

1.1只在ID上进行约束

主键约束即为primary key.

接下来使用具体实例来分析其用法，通过创建数据表并向其中插入元素，再次向数据表中插入相同name的元素。

第二次插入的结果显示插入失败系统报错，这是因为数据表中存在数据主键之间的约束，一旦将某个元素设置为主键约束，则该元素只能在当前数据表中出现一次：

1.2在ID和name上都进行约束

primary key(id, name)为联合主键约束，只要这两个主键值不同时相同即可。

2.自增约束

primary key auto_increment自增约束，可以自动管控主键值的增加，搭配主键约束一起使用，其特性为：不重复且不为空自增。

2.1在创建表的时候没有创建主键约束的处理办法：通过add添加

修改表结构，通过alter table user4 add primary key (id);添加主键约束。

2.2通过drop删除主键约束

通过alter table user4 drop primary key;删除主键约束。

2.3使用modify修改字段，添加约束

alter table user4 modify id int primary key;

3.唯一约束

3.1约束修饰的字段的值不可以重复alter table user5 add unique (name);

3.2创建表时直接添加唯一约束

3.3添加多个唯一约束unique (id,name)

表示两个键在一起不重复就行，ID和name组合在一起不重复就是OK的。

3.4删除唯一约束

alter table user6 drop index id;

3.5通过modify来添加

alter table user6 modify name varchar(20) unique;

3.6总结

  1. 建表的时候就添加约束
  2. 可以使用 alter 。。。 add 。。。
  3. alter 。。。 modify 。。。
  4. 删除 alter 。。。 drop 。。。

4.非空约束

修饰的字段不能为空not null

5.默认约束

当我们插入字段值的时候，如果没有传值，就会使用默认值。如果传值将会使用新传入的值，而覆盖掉默认的值。
