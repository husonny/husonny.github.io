Termux+Hexo搭建Github个人博客

"Hello Hexo ."

Posted by Sonny on 2019-08-22

Termux + Hexo 可以让你在 Android 移动设备上部署自己的GitHub个人博客，同时可以随时随地撰写博文。

1.前期准备

（1）注册GitHub账号

进入GitHub官网进行注册，该注册步骤此处不再赘述。

（2）在GitHub上创建个人仓库

登陆个人GitHub主页，点击Your repositories进入仓库列表，点击New以此来创建新的仓库。

输入仓库名称，此处名称必须是your-github-username.github.io，以此按照提示完成。

（3）安装Termux高级终端

进入Termux 官网，官网放出安装方式有两种，从Google paly（需要翻墙）或者F-Driod进行安装。此处给出百度网盘安装包链接Termux APK 。

（4）购买域名（可选）

可以将自己的个人博客域名设置为购买的域名，比如我购买的域名是easy4me.cn，并且部署到我的个人博客上了，现在可以通过该域名访问我的个人博客，域名地址不再是 husonny.github.io 了。

2.安装git

（在以下2-4步可以翻墙挂VPN来使下载安装速度加快）

$ pkg install git

3.安装Node.js

Hexo框架是需要Node.js来支持的，新版的Node.js是已经集成了npm包管理工具，因此npm服务是不许要额外安装的。

$ pkg install nodejs

4.安装Taobao源

Termux自带的是国外的源，国内用户使用起来不方便，下载很慢，安装国内的源还是很有必要的，此处选用的是Taobao源，当然也可以使用国内其他的源。

$ npm install -g cnpm --registory=https://registory.npm.taobao.org

5.安装Hexo框架

在前面已经安装过Taobao源，现在可以直接使用cnpm来安装软件

$ cnpm install -g hexo-cli

6.本地部署

在根目录创建blog文件夹（该文件夹名称可自定，后期能快速找到就行），并用Hexo生成网站框架

在浏览器输入http://localhost:4000 可以查看到在本地部署好的默认网页

7.部署到远程GitHub

(1)安装远程部署插件

$ cnpm install --save hexo-deployer-git

8.添加个人域名（可选）

(1)配置CNAME文件

进入GitHub个人仓库，点击Create new file新建一个文件，将文件命名为CNAME，编辑该文件，并填写需要绑定的域名，如easy4me.cn，保存。

(2)GitHub设置绑定域名

进入GitHub设置，在Custom domain选项下添加需要绑定的域名，保存。

(3)添加域名解析

在Termux终端中，使用ping命令可以检查网络是否连通，同时也可以简单的获取目标地址的IP。

其中 xx.xxx.xxx.xx 显示的便是IP，获取该IP后进入云服务后台，一般为阿里云或者腾讯云，在控制台找到进入域名解析列表，添加域名解析DNS，修改记录值，填入上述步骤中所获取的IP，重新启动域名解析即可。
